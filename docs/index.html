Build your own Mirador Instance

This is a showcase of how to play around building your own mirador instance. Wanna see how? Have a look in the repo.

  1. A basic viewer couldn't be simpler just a few lines of html served from a web server and this is it. Have a look at the html . Wanna see how looks? There you go.
  2. Mirador viewers can be configured easily with a wide range of options ( see the Config Recipes )
    I found it especially useful to set things up via query params. This way one viewer can serve many use cases with just some query params set differently.
    • Like a minimal viewer for situations like an exhibition showing a work on a tablet device where the user should stick to a manifest and only leaf some pages and zoom in and out.
      In this case we only need to add to query params like that:
      ?page=5&manifest=https://example.com/your-desired-manifest.json
      This is how it turns out.
    • You can modify a lot more than that. If you have two images that should be shown like an opened book you can select a the view. If the reader should be presented with the table of contents in sidebar.
      This would be something like:
      ?page=15&view=book&sideBarPanel=canvas&sideBarOpen=true&manifest=[...]
      Take a look .

    These are the options I implemented here:

    • manifest: this sets the loaded manifest as an URL.
    • page: (default: 1) the number of the page to be opened at first
    • view : (default: single) select whether to show an single image (single), two next to each other (book) or a grid of small thumbnails (gallery)
    • language: (default: en) select the language of the UI at startup. ( e.g. en, de)
    • allowFullscreen: (default: false ) set whether a button should be shown to switch to fullscreen mode if needed allowFullscreen=false
    • allowTopMenuButton (default true): decide wheter to show a button to switch the view and plugins if needed allowTopMenuButton=false
    • sideBarPanel: decide whether a side bar panel is available at all. Standard options: info, attribution, canvas, annotations. turned off by default if needed sideBarPanel=info
    • sideBarOpen : set whether the side bar panel should be opened on startup. (Only effective when a sideBarPanel option is set) turned off by default if needed sideBarOpen=true
    • showZoomControls: sets whether buttons for zooming should be displayed. turned on by default if needed showZoomControls=false
    • showPagination: sets whether buttons for leafing should be displayed. turned on by default if needed showPagination=false
    • showPageInfo: sets whether the page info on the buttom should be shown (page number and label) turned on by default if needed showPageInfo=false
